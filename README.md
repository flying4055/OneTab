# OneTab

一个高度可定制、极简风格的浏览器新标签页导航扩展，基于 **React + Material-UI (MUI) + Vite** 构建，旨在提供沉浸式的书签检索与管理体验。

## 核心功能

1. **书签管理**：支持快速新增、编辑、删除书签（包含自定义图标、背景色、文字颜色）。
2. **全局搜索**：集成各大搜索引擎，同时支持防抖的本地书签模糊匹配（按名称与域名）。
3. **沉浸式 UI**：全局应用暗黑毛玻璃（Glassmorphism）质感，提供精致的视觉体验。
4. **个性化壁纸**：支持上传本地图片作为新标签页背景。
5. **数据导入导出**：可将当前书签配置导出为 JSON 文件，或从 JSON 文件导入，方便数据备份与迁移。
6. **纯本地化**：所有数据及配置（如 `nav_data_v1`、壁纸等）均存储于浏览器本地 `localStorage`，安全且隐私。

## 技术架构

- **扩展类型**：Chrome/Edge MV3 新标签页覆盖 (`chrome_url_overrides`)。
- **前端框架**：React 19 + Material-UI v5。
- **构建工具**：Vite（极速冷启动与 HMR）。
- **包管理器**：pnpm。

## 目录结构

- `src/`：前端核心源代码。
  - `main.jsx` / `App.jsx`：React 渲染入口与状态容器。
  - `components/`：按业务领域划分的组件（如 `sites` 标签网格、`search` 搜索栏、`settings` 设置面板、`wallpaper` 壁纸管理）。
  - `theme/`：MUI 主题定制配置。
- `manifest.json`：MV3 扩展配置文件。
- `nav.json`：书签的初始种子数据。

## 安装与使用

### 开发环境预览

1. 确保已安装 [Node.js](https://nodejs.org/) 与 `pnpm`。
2. 克隆或下载本项目至本地。
3. 在项目根目录执行以下命令：
   ```bash
   pnpm install
   pnpm run dev
   ```
4. 在浏览器中访问 `http://localhost:5173` 即可进行独立页面的开发调试。

### 浏览器扩展部署

1. 在项目根目录运行构建命令生成最终产物：
   ```bash
   pnpm run build
   ```
   > 构建完成后，项目根目录会生成一个 `dist` 文件夹。
2. 打开 Chrome 或 Edge 浏览器的扩展管理页 (`chrome://extensions` 或 `edge://extensions`)。
3. 开启右上角的 **开发者模式**。
4. 点击 **加载已解压的扩展程序**。
5. 选择本项目刚刚生成的 `dist` 文件夹。
6. 打开一个新的浏览器标签页，即可体验全新的 OneTab。

## 交互说明

- **搜索与跳转**：在居中的搜索栏中输入内容，下拉列表将呈现玻璃质感，优先匹配本地书签名称和网址域名（支持键盘上下键选中和回车跳转）。若未选中本地书签，回车则使用默认搜索引擎进行网络搜索。
- **书签操作**：在书签网格上右键点击任意书签图标，即可在弹出的菜单中选择「编辑」或「删除」。
- **全局设置**：点击右下角的齿轮图标打开设置面板，可以在此修改「是否在新标签页打开链接」、进行数据的导入/导出，或打开「壁纸管理」弹窗。
- **添加书签**：点击书签网格最后的 `+` 号按钮即可唤出新增书签的表单。

## 数据格式与存储

初始数据读取自根目录的 `nav.json`，结构如下：
```json
[
  {
    "id": "1",
    "label": "主页",
    "items": [
      {
        "id": "site-xxx",
        "name": "哔哩哔哩",
        "url": "https://www.bilibili.com",
        "src": "可选的自定义图标URL或Base64",
        "bg-color": "#ffffff"
      }
    ]
  }
]
```

用户在界面上进行的所有修改均会序列化并持久化存储至 `localStorage` 中的 `nav_data_v1`。

## 常见问题

**Q: 为什么我修改了 `nav.json`，但是页面上的书签没有更新？**  
A: 当系统检测到 `localStorage` 中已存在 `nav_data_v1` 时，会优先读取用户的本地存储。若想重新应用 `nav.json` 的数据，请打开控制台并清除 `localStorage`，然后刷新页面即可。
